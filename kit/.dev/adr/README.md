# .dev/adr/ — Architecture Decision Records

One `.md` per decision: `ADR-NNN-short-title.md`. Each has: Date · Status ·
Context · Decision · Consequences · Alternatives Considered · Related · Notes.

**This directory is a source of truth.** Before changing anything architectural
(stack, layering, boundaries, cross-cutting rules), grep here for a relevant
ADR. If your change contradicts an accepted ADR, either align with it or write a
new ADR that **supersedes** it — never silently violate. Keep an index of
accepted ADRs in the project's `CLAUDE.md`.

Copy `ADR-000-record-architecture-decisions.md` as the starting template.
